VMS agent collector (Vero Metric Collector) chạy trực tiếp tại host hoặc customer site để thu thập dữ liệu giám sát (system metrics, process, logs, connections, port status) và gửi về VMS Master qua HTTPS.

<Info>
**Hỗ trợ đa nền tảng**: VMS Agent chạy trực tiếp trên **Linux** (systemd) và **Windows**, ngoài ra hỗ trợ container hóa qua Docker/Kubernetes.
</Info>

## Các đầu mục chuẩn bị (Agent)

| Đầu mục | Yêu cầu chuẩn bị |
|---|---|
| OS | **Linux** (systemd) hoặc **Windows** |
| Kết nối | Không cần mở inbound. Outbound HTTPS tới `INGRESS_URL` để gửi metric/log và nhận config, và tới Dashboard để tải binary agent |
| API Key | API key hợp lệ sinh từ VMS Master, nhúng sẵn trong script cài |

## Bước 0 — Chuẩn bị script và binary (trên Dashboard)

Quy trình cài đặt dùng script `agent.sh` (Linux) hoặc `agent.ps1` (Windows). Mỗi script nhúng cấu hình trong khối `CONFIG` và tự tải binary agent về.

<Steps>
  <Step title="1. Lấy script và binary từ File Storage">
    Trên VMS Dashboard, mở **File Storage**, upload (hoặc lấy sẵn) `agent.sh`/`agent.ps1` và archive binary agent. Dùng nút **Copy link** (alias) để lấy link luôn trỏ tới bản mới nhất.
  </Step>
  <Step title="2. Sửa khối CONFIG trong script">
    Mở script và điền các giá trị trong khối `CONFIG`:
    ```bash
    INGRESS_URL="https://ingest.CUSTOMER_DOMAIN"   # endpoint VMS Master
    API_KEY="<API key từ VMS Master>"
    BIN_URL="<alias link archive binary trên File Storage>"
    CA_URL=""                                       # link CA nội bộ; để trống = bỏ qua xác thực TLS
    ```
    Lưu và upload lại script lên File Storage để tái sử dụng cho nhiều máy.
  </Step>
</Steps>

## Quy trình triển khai trên Linux

Quy trình cài đặt theo cấu trúc **Thực hiện (Step) - Kiểm tra (Check) - Kết quả (Result)**:

<Steps>
  <Step title="1. Chạy script cài đặt">
    - **Step**: Trên host cần giám sát, chạy script đã cấu hình (không cần sudo):
      ```bash
      curl -fsSL "<agent.sh link>" | sh
      ```
      Script tự động tải binary, ghi config, cài service, enable + start, rồi tự verify đăng ký.
    - **Check**: Script in tiến trình `Step 1/6 ... 6/6` và kết thúc bằng banner kết quả.
    - **Result**: `SUCCESS (verified): vms-monitor-agent <version> installed`.
  </Step>

  <Step title="2. Kiểm tra service">
    - **Step**: Xem trạng thái service:
      ```bash
      systemctl --user status vms-monitor-agent --no-pager
      ```
    - **Check**: Trạng thái hiển thị `active (running)`.
    - **Result**: Binary tại `~/.local/bin/vms-monitor-agent`, config tại `~/.config/vms-monitor-agent/agent.env`.
  </Step>

  <Step title="3. Kiểm tra logs kết nối">
    - **Step**: Xem log thời gian thực:
      ```bash
      journalctl --user -u vms-monitor-agent -f
      ```
    - **Check**: Tìm dòng log `agent started` kèm `agentKey`, `probes`, `configVersion`.
    - **Result**: Log dạng JSON xuất hiện, ví dụ:
      ```json
      {"level":"INFO","msg":"agent started","agentKey":"web-01/appuser","configVersion":12,"probes":8}
      ```
  </Step>
</Steps>

**Cập nhật**: chạy lại đúng lệnh cài; script tự so version và hỏi trước khi thay binary (đặt `VMS_ASSUME_YES=1` để cập nhật không tương tác).
**Gỡ cài**: `curl -fsSL "<agent.sh link>" | sh -s -- uninstall` (thêm `--purge` để xoá cả state).

## Quy trình triển khai trên Windows

Quy trình cài đặt theo cấu trúc **Thực hiện (Step) - Kiểm tra (Check) - Kết quả (Result)**:

<Steps>
  <Step title="1. Chạy script cài đặt">
    - **Step**: Mở PowerShell và chạy (không cần quyền Administrator):
      ```powershell
      irm "<agent.ps1 link>" | iex
      ```
      Script tự tải + giải nén binary, ghi config, đăng ký auto-start, start agent, rồi verify.
    - **Check**: Script in tiến trình `Step 1/6 ... 6/6` và kết thúc bằng banner.
    - **Result**: `SUCCESS (verified): vms-monitor-agent <version> installed`.
  </Step>

  <Step title="2. Kiểm tra trạng thái agent">
    - **Step**: Kiểm tra task auto-start và tiến trình agent:
      ```powershell
      Get-ScheduledTask -TaskName vms-monitor-agent
      Get-Process vms-monitor-agent
      ```
    - **Check**: Task tồn tại và có process agent đang chạy.
    - **Result**: Binary tại `%LOCALAPPDATA%\vms-monitor-agent\vms-monitor-agent.exe`.
  </Step>

  <Step title="3. Kiểm tra version">
    - **Step**: In thông tin phiên bản:
      ```powershell
      & "$env:LOCALAPPDATA\vms-monitor-agent\vms-monitor-agent.exe" version
      ```
    - **Check**: In ra version, commit và build date.
    - **Result**: Xác nhận đúng phiên bản vừa cài.
  </Step>
</Steps>

**Cập nhật**: chạy lại `irm "<agent.ps1 link>" | iex`.
**Gỡ cài**: `& ([scriptblock]::Create((irm "<agent.ps1 link>"))) uninstall` (thêm `-Purge` để xoá cả thư mục cài).

## Troubleshooting thường gặp

| Triệu chứng | Nguyên nhân thường gặp | Cách xử lý |
|---|---|---|
| Agent start fail ngay | Thiếu ingress URL/API key | Kiểm tra và sửa khối CONFIG trong script rồi chạy lại |
| `401`/auth blocked | API key sai hoặc bị rotate | Lấy key mới từ VMS Master và chạy lại script |
| `x509 unknown authority` | Ingress dùng private CA | Khai báo `CA_URL` trỏ tới CA nội bộ (script set `VMS_TLS_CA_FILE`) |
| Ping chuyển TCP fallback | Thiếu raw ICMP permission | Cấp `CAP_NET_RAW` hoặc chấp nhận TCP fallback |
| Connection/process thiếu process name | Thiếu quyền đọc process attribution | Chạy với quyền phù hợp hoặc cấp `CAP_SYS_PTRACE` theo policy |
| Container probe không chạy | Binary mặc định không kèm probe container | Dùng bản build `vms_heavy` và mount socket read-only theo policy |
| K8s probe permission missing | Thiếu ServiceAccount/RBAC | Cấp get/list workload/pod/node và metrics-server nếu `collectStats` |
