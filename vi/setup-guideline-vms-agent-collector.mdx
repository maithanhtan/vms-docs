VMS agent collector (Vero Metric Collector) chạy trực tiếp tại host hoặc customer site để thu thập dữ liệu giám sát (system metrics, process, logs, connections, port status) và gửi về VMS Master qua HTTPS.

<Info>
**Hỗ trợ đa nền tảng**: VMS Agent chạy trực tiếp trên **Linux** và **Windows**, ngoài ra hỗ trợ container hóa qua Docker/Kubernetes.
</Info>

## Các đầu mục chuẩn bị (Agent)

| Đầu mục | Yêu cầu chuẩn bị |
|---|---|
| OS | **Linux** hoặc **Windows** |
| Kết nối | Không cần mở inbound. Outbound HTTPS tới `INGRESS_URL` để gửi metric/log và nhận config, và tới Dashboard để tải binary agent |
| API Key | API key hợp lệ sinh từ VMS Master, nhúng sẵn trong script cài |
| TLS Mode | Public CA, file CA nội bộ, hoặc bỏ qua xác thực cert (mặc định khi để trống `CA_URL`) |

## Bước 0 — Chuẩn bị script và binary (trên Dashboard)

Quy trình cài đặt dùng script `agent.sh` (Linux) hoặc `agent.ps1` (Windows). Mỗi script nhúng cấu hình trong khối `CONFIG` và tự tải binary agent về.

<Steps>
  <Step title="1. Lấy script và binary từ File Storage">
    Trên VMS Dashboard, mở **File Storage**, upload (hoặc lấy sẵn) `agent.sh`/`agent.ps1` và archive binary agent. Dùng nút **Copy link** (alias) để lấy link luôn trỏ tới bản mới nhất.

    ![Danh sách file agent trong File Storage](/images/agent-install/08-file-storage.png)
  </Step>
  <Step title="2. Sửa khối CONFIG trong script">
    Mở script và điền các giá trị trong khối `CONFIG`:
    ```bash
    INGRESS_URL="https://ingest.CUSTOMER_DOMAIN"   # endpoint VMS Master
    API_KEY="<API key từ VMS Master>"
    BIN_URL="<alias link archive binary trên File Storage>"
    CA_URL=""                                       # link CA nội bộ; để trống = bỏ qua xác thực TLS
    ```
    Lưu và upload lại script lên File Storage để tái sử dụng cho nhiều máy.
  </Step>
</Steps>

## Quy trình triển khai trên Linux

Quy trình cài đặt theo cấu trúc **Thực hiện (Step) - Kiểm tra (Check) - Kết quả (Result)**:

<Steps>
  <Step title="1. Chạy script cài đặt">
    - **Step**: Trên host cần giám sát, chạy script đã cấu hình (không cần sudo):
      ```bash
      curl -fsSL "<agent.sh link>" | sh
      ```
      Script tự tải binary, ghi config, cài đặt, khởi động và tự verify đăng ký.
    - **Check**: Script in tiến trình `Step 1/6 ... 6/6` và kết thúc bằng banner kết quả.
    - **Result**: `SUCCESS (verified): vms-monitor-agent <version> installed`.

    ![Kết quả chạy script cài đặt agent trên Linux](/images/agent-install/01-install.png)
  </Step>

  <Step title="2. Kiểm tra service">
    - **Step**: Xem trạng thái service:
      ```bash
      systemctl --user status vms-monitor-agent --no-pager
      ```
    - **Check**: Trạng thái hiển thị `active (running)`.
    - **Result**: Binary tại `~/.local/bin/vms-monitor-agent`, config tại `~/.config/vms-monitor-agent/agent.env`.

    ![Trạng thái service vms-monitor-agent đang chạy](/images/agent-install/02-service-status.png)
  </Step>

  <Step title="3. Kiểm tra logs kết nối">
    - **Step**: Xem log thời gian thực:
      ```bash
      journalctl --user -u vms-monitor-agent -f
      ```
    - **Check**: Tìm dòng log `agent started` kèm `agentKey`, `probes`, `configVersion`.
    - **Result**: Log dạng JSON xuất hiện, ví dụ:
      ```json
      {"level":"INFO","msg":"agent started","agentKey":"web-01/appuser","configVersion":12,"probes":8}
      ```

    ![Log JSON "agent started" trên Linux](/images/agent-install/03-logs.png)
  </Step>
</Steps>

Sau khi cài xong, agent tự đăng ký và hiện trong **Agent Config (FLEET)** trên Dashboard với trạng thái online (chấm xanh), kèm version, IP và số probe:

![Agent online trong FLEET trên Dashboard](/images/agent-install/09-agent-online.png)

**Cập nhật**: chạy lại đúng lệnh cài; script tự so version và hỏi trước khi thay binary (đặt `VMS_ASSUME_YES=1` để cập nhật không tương tác). Khi host đã ở bản mới nhất, script chỉ kiểm tra sức khỏe rồi báo `already current`:

![Chạy lại lệnh cài khi agent đã ở bản mới nhất](/images/agent-install/07-update.png)

**Gỡ cài** — chạy thẳng bằng binary (khuyến nghị), hoặc tải lại script:

```bash
# Gỡ ngay trên máy bằng binary; thêm --purge để xoá cả state:
vms-monitor-agent uninstall
# Hoặc, nếu host chưa có lệnh này, tải lại script (giống "Run command" trên Dashboard):
curl -fsSL "<agent.sh link>" | sh -s -- uninstall
```

![Gỡ agent bằng lệnh vms-monitor-agent uninstall](/images/agent-install/06-uninstall.png)

## Quy trình triển khai trên Windows

Quy trình cài đặt theo cấu trúc **Thực hiện (Step) - Kiểm tra (Check) - Kết quả (Result)**:

<Steps>
  <Step title="1. Chạy script cài đặt">
    - **Step**: Mở PowerShell và chạy (không cần quyền Administrator):
      ```powershell
      irm "<agent.ps1 link>" | iex
      ```
      Script tự tải + giải nén binary, ghi config, cài đặt, khởi động và verify.
    - **Check**: Script in tiến trình `Step 1/6 ... 6/6` và kết thúc bằng banner.
    - **Result**: `SUCCESS (verified): vms-monitor-agent <version> installed`.
  </Step>

  <Step title="2. Kiểm tra trạng thái agent">
    - **Step**: Kiểm tra task auto-start và tiến trình agent:
      ```powershell
      Get-ScheduledTask -TaskName vms-monitor-agent
      Get-Process vms-monitor-agent
      ```
    - **Check**: Task tồn tại và có process agent đang chạy.
    - **Result**: Binary tại `%LOCALAPPDATA%\vms-monitor-agent\vms-monitor-agent.exe`.
  </Step>

  <Step title="3. Kiểm tra logs / version">
    - **Step**: In thông tin phiên bản và kiểm tra agent đã đẩy dữ liệu lên Dashboard:
      ```powershell
      & "$env:LOCALAPPDATA\vms-monitor-agent\vms-monitor-agent.exe" version
      ```
    - **Check**: In ra version, commit và build date; agent xuất hiện trong danh sách trên Dashboard.
    - **Result**: Đúng phiên bản vừa cài và agent đang online.
  </Step>
</Steps>

**Cập nhật**: chạy lại lệnh cài ở trên (`irm "<agent.ps1 link>" | iex`).

**Gỡ cài** — chạy thẳng bằng binary (khuyến nghị), hoặc tải lại script đúng như nút **Copy** trên Dashboard:

```powershell
# Gỡ ngay trên máy bằng binary; thêm --purge để xoá cả thư mục cài:
vms-monitor-agent uninstall
# Hoặc tải lại script (giống "Run command" trên Dashboard):
& ([scriptblock]::Create((irm "<agent.ps1 link>"))) uninstall
```

## Quản lý agent bằng dòng lệnh

Sau khi cài, thư mục chứa binary được thêm vào `PATH` nên chỉ cần **mở terminal/PowerShell mới** là gõ được lệnh `vms-monitor-agent` theo tên trần (không cần `sudo`/Administrator). Script cài cũng **in sẵn danh sách lệnh này ở cuối** khi cài xong. Các lệnh điều khiển đúng cơ chế per-user mà script đã dựng: systemd `--user` (Linux) và Scheduled Task (Windows).

| Lệnh | Tác dụng |
|---|---|
| `vms-monitor-agent status` | Xem trạng thái: đã cài / đang chạy / auto-start / version |
| `vms-monitor-agent start` | Khởi động và bật tự chạy lại khi đăng nhập/khởi động máy |
| `vms-monitor-agent stop` | Dừng **hẳn** và tắt auto-start — agent **sẽ không tự dậy lại** |
| `vms-monitor-agent restart` | Khởi động lại, giữ nguyên auto-start |
| `vms-monitor-agent uninstall` | Tự gỡ agent (thêm `--purge` để xoá cả state) |
| `vms-monitor-agent version` | In version, commit, build date |
| `vms-monitor-agent help` | In hướng dẫn sử dụng đầy đủ |

Lệnh `status` cho biết agent đã cài / đang chạy / auto-start / version:

![Kết quả lệnh vms-monitor-agent status](/images/agent-install/04-agent-status.png)

Minh hoạ `stop` (dừng hẳn) rồi `start` lại — chú ý `running` và `auto-start` đổi trạng thái theo:

![Vòng đời stop và start agent](/images/agent-install/05-lifecycle.png)

<Info>
**`stop` là dừng hẳn**: nó vừa dừng tiến trình vừa tắt auto-start, nên agent **không tự chạy lại sau reboot hay đăng nhập lại** (kể cả khi process bị kill). Muốn chạy lại dùng `vms-monitor-agent start`; muốn gỡ hẳn dùng `vms-monitor-agent uninstall`.

Nếu gõ tên trần mà shell báo `command not found` (ví dụ `bash: vms-monitor-agent: command not found`; trên PowerShell là `The term 'vms-monitor-agent' is not recognized...`), hãy mở shell mới, hoặc dùng đường dẫn đầy đủ (`~/.local/bin/vms-monitor-agent` trên Linux, `%LOCALAPPDATA%\vms-monitor-agent\vms-monitor-agent.exe` trên Windows). Nếu agent in `unknown command "status"`, host đang chạy bản agent cũ chưa hỗ trợ lệnh quản lý — hãy cập nhật trước (chạy lại lệnh cài).
</Info>

## Troubleshooting thường gặp

| Triệu chứng | Nguyên nhân thường gặp | Cách xử lý |
|---|---|---|
| Agent start fail ngay | Thiếu ingress URL/API key | Kiểm tra và sửa khối CONFIG trong script rồi chạy lại |
| `401`/auth blocked | API key sai hoặc bị rotate | Lấy key mới từ VMS Master và chạy lại script |
| `x509 unknown authority` | Ingress dùng private CA | Khai báo `CA_URL` trỏ tới CA nội bộ (script set `VMS_TLS_CA_FILE`) |
| Ping chuyển TCP fallback | Thiếu raw ICMP permission | Cấp `CAP_NET_RAW` hoặc chấp nhận TCP fallback |
| Connection/process thiếu process name | Thiếu quyền đọc process attribution | Chạy với quyền phù hợp hoặc cấp `CAP_SYS_PTRACE` theo policy |
| Container probe không chạy | Binary mặc định không kèm probe container | Dùng bản build `vms_heavy` và mount socket read-only theo policy |
| K8s probe permission missing | Thiếu ServiceAccount/RBAC | Cấp get/list workload/pod/node và metrics-server nếu `collectStats` |
